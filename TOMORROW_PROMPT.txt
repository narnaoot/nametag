Hi! We're continuing work on the Nametag app — a React + Vite + Capacitor app that will run on iPhone.
The repo is at https://github.com/narnaoot/nametag and is already cloned locally.

Before we add new features, let's do some refactoring. Here are the priorities, in order:

1. FIX TOKEN SYNC BUG (most important)
   - AuthContext.jsx saves the token to @capacitor/preferences
   - api.js reads the token from localStorage
   - These two can diverge and cause silent auth failures
   - Fix: api.js should get the token from @capacitor/preferences too,
     or AuthContext should expose it so api.js can use it consistently

2. EXTRACT A SHARED <PageHeader /> COMPONENT
   - AuthPage, ProfilePage, and GridPage all repeat the same header
     (logo, red color #E63946, Caveat font, sign-out button)
   - Extract to a reusable PageHeader component in src/components/

3. MOVE SHARED CONSTANTS TO ONE FILE
   - BANNER_COLORS is defined in both DesignE.jsx and ProfilePage.jsx
   - PRONOUN_OPTIONS, STICKER_OPTIONS, RADIUS_OPTIONS only live in ProfilePage.jsx
   - Create src/constants.js with all of these exported from one place

4. FIX SILENT ERROR SWALLOWING
   - Several catch blocks in GridPage.jsx and ProfilePage.jsx are empty
     or swallow errors with no logging
   - At minimum add console.error() so bugs are debuggable

5. EXTRACT A useNearbyPeople() HOOK
   - GridPage.jsx manages location sharing, profile loading, and nearby
     people polling all in one component
   - Extract this logic to src/hooks/useNearbyPeople.js

After refactoring, we'll want to work on:
- Better loading/error states in the UI
- Input validation on forms
- Possibly picking one of the Design variants (A-E) to keep and deleting the rest

The app uses:
- React 19 + Vite (client/)
- Express + MongoDB (server/)
- Capacitor for iOS (capacitor.config.ts)
- Tailwind CSS + inline styles (mixed — part of what we're cleaning up)
- @capacitor/geolocation, @capacitor/camera, @capacitor/preferences

Let's start with item 1 (the token bug) and work down the list.
