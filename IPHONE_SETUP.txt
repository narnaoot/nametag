NAMETAG - IPHONE APP SETUP GUIDE
=================================

PREREQUISITES (do these once on your Mac)
------------------------------------------
1. Install Xcode from the Mac App Store (free, ~10GB)
2. Open Xcode, go to Xcode > Settings > Accounts, sign in with your Apple ID
3. Install Node.js if not already: https://nodejs.org
4. Install CocoaPods: sudo gem install cocoapods


STEP 1: DEPLOY THE BACKEND TO RENDER
--------------------------------------
1. Go to https://render.com and create a free account
2. Click "New" > "Web Service"
3. Connect your GitHub account and select the nametag repo
4. Set these settings:
   - Root Directory: server
   - Build Command: npm install
   - Start Command: node index.js
   - Environment: Node
5. Add environment variables in Render dashboard:
   - MONGODB_URI = (your MongoDB connection string)
   - JWT_SECRET = (any long random string)
   - NODE_ENV = production
6. Click "Create Web Service" and wait for it to deploy
7. Copy the URL Render gives you (e.g. https://nametag-api.onrender.com)


STEP 2: UPDATE THE FRONTEND CONFIG
------------------------------------
1. Open client/.env.production in your code editor
2. Replace the placeholder with your Render URL:
   VITE_API_URL=https://your-actual-render-url.onrender.com
3. Save the file, commit and push to GitHub


STEP 3: SET UP THE PROJECT ON YOUR MAC
----------------------------------------
1. Clone the repo on your Mac:
   git clone https://github.com/narnaoot/nametag.git
   cd nametag

2. Install frontend dependencies and build:
   cd client
   npm install
   npm run build
   cd ..

3. Add the iOS platform:
   cd client
   npx cap add ios

4. Sync the web build to iOS:
   npx cap sync ios


STEP 4: OPEN IN XCODE
-----------------------
1. From the client folder, run:
   npx cap open ios

   (This opens Xcode automatically)

2. In Xcode, click on "App" in the left sidebar (the top-level project)
3. Under "Signing & Capabilities":
   - Check "Automatically manage signing"
   - Set Team to your Apple ID
   - Bundle Identifier is already set to: com.nametag.app


STEP 5: TEST ON YOUR IPHONE (no developer account needed)
----------------------------------------------------------
1. Connect your iPhone to your Mac with a USB cable
2. Trust the computer when prompted on your iPhone
3. In Xcode, select your iPhone from the device dropdown at the top
4. Click the Play button (triangle) to build and install
5. On your iPhone: Settings > General > VPN & Device Management
   > Trust your Apple ID email > Trust
6. Open the Nametag app on your iPhone


STEP 6: PUBLISH TO THE APP STORE (when ready)
-----------------------------------------------
1. You need a paid Apple Developer account ($99/year):
   https://developer.apple.com/programs/enroll/
2. In Xcode: Product > Archive
3. Click "Distribute App" > "App Store Connect"
4. Follow the prompts to upload
5. Go to https://appstoreconnect.apple.com
6. Fill in app description, screenshots, category
7. Submit for Apple Review (usually takes 1-3 days)


PERMISSIONS (already configured in the app)
---------------------------------------------
- Location: used for the map/grid feature
- Camera: used for profile photo
These will prompt the user automatically on first use.


UPDATING THE APP AFTER CODE CHANGES
-------------------------------------
Each time you change the frontend code:
1. cd client
2. npm run build
3. npx cap sync ios
4. Rebuild in Xcode (Cmd+R)


TROUBLESHOOTING
----------------
- "No signing certificate" error: Make sure you're signed into Xcode with your Apple ID
- App crashes on launch: Check the Render backend is running and .env.production URL is correct
- Location/camera not working: Delete app from iPhone and reinstall to re-trigger permission prompts
- Pod install errors: cd ios && pod install --repo-update
